Explain git rebase vs merge

explain git
Normal agent
The main difference between git rebase and git merge is that merge creates a new commit that combines the histories of two branches, preserving the complete history. Rebase, on the other hand, rewrites the commit history by applying commits from one branch onto the tip of another.
~55 tokens
Leanman
Merge: new commit, preserves history. Rebase: rewrites commits onto tip, linear history. Use merge for public branches, rebase for local cleanup.
